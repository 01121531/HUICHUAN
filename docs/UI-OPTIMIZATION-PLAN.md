# UI 视觉与交互优化方案

## 1. 文档目标

本方案基于当前前端页面的实际桌面端与移动端渲染效果，对主页、登录注册、找回密码、模型广场、关于页、错误页及后台页面体系进行统一规划，重点优化：

- 品牌视觉一致性
- 响应式布局
- 页面信息层级
- 动画与微交互
- 空状态、加载状态和错误状态
- 可访问性与性能

本方案不改变现有业务流程和 API 契约，优先通过前端组件、设计 Token、布局和动效系统完成升级。

## 3. 整体问题

### 3.1 品牌视觉不统一

当前主要存在三套视觉语言：

1. 主页使用紫色渐变、科技网格、粒子和 `TUO` + `SHANG` 组合。
2. 登录注册页面使用旧圆形 Logo、全小写站点名、纯白背景和蓝色按钮。
3. 模型广场、关于页等公共页面使用旧公共导航和蓝绿色背景光晕。

建议建立统一品牌组件：

- 标准字标：`TUO` + 渐变 `SHANG`
- 强调字标：`TUOSHANG AI`
- 紧凑空间：仅显示图形标
- 主色：Indigo → Violet → Fuchsia
- 蓝色降级为信息色，不再作为主要 CTA 色
- 抽取统一的 `BrandMark`、`BrandWordmark` 和公共导航外观

项目治理要求保护的 New API、QuantumNous 等项目归属和许可证信息必须保留，建议集中放置在关于页和页脚的合规区域，不与主要产品品牌层级竞争。

### 3.2 页面体系缺少统一的布局节奏

- 主页使用较丰富的营销布局。
- 认证页面留白过大且缺乏视觉容器。
- 模型广场信息密度较高。
- 关于页和错误页内容过少，显得像未完成页面。
- 不同页面的最大宽度、区块间距、按钮样式和背景处理没有统一规则。

建议统一：

- 公共内容宽度：`1280px`
- 数据密集页面最大宽度：`1600px–1800px`
- 移动端水平边距：`20px`
- 桌面端区块间距：`96px–120px`
- 移动端区块间距：`64px–80px`
- 卡片圆角：`16px`、`24px` 两级
- 主要 CTA 高度：`48px`
- 表单控件高度：`44px–48px`

## 4. 主页优化方案

### 4.1 当前优点

- 首屏标题层级清晰。
- 紫色光晕、背景网格、状态点和终端演示具有较强品牌识别度。
- 导航收缩和滚动进度动画表现良好。
- 桌面端和移动端均无明显横向溢出。

### 4.2 当前问题

- 页面纵向较长，移动端连续阅读成本较高。
- 多个能力和场景卡片在移动端连续纵向排列，节奏较单一。
- 首屏动画较丰富，中后段动画类型相对重复。
- API Console 在窄屏主要依赖横向滚动。
- 主页导航与其他公共页面导航并非同一套视觉实现。

### 4.3 布局优化

1. 移动端将“开发者”和“企业”内容改为双 Tab 或横向 Snap 卡片。
2. 应用场景在桌面端保持 3×2 网格，移动端改为横向滑动卡片。
3. FAQ 默认显示 4 项，其余通过“查看更多”展开。
4. 首屏桌面高度控制在约 `900px–980px`，让终端底部在常见屏幕中可见。
5. API Console 在移动端展示精简请求，完整请求通过展开或复制按钮查看。
6. 统一主页与其他公共页面的品牌标识、主题按钮和移动菜单样式。

### 4.4 动画优化

- Hero 元素使用 `60ms–80ms` 间隔错峰入场。
- 终端保留扫描线，同时增加路由节点依次点亮效果。
- 模型列表继续循环滚动，鼠标悬停时暂停。
- 能力卡片进入视口时使用 `translateY(20px) + opacity`。
- 能力卡片 Hover 时上浮 `4px`，边框轻微染紫，图标旋转不超过 `3deg`。
- 场景卡片每行最多错峰 3 个，避免后续卡片等待过长。
- CTA 背景使用低速渐变位移，不使用高速流光。

## 5. 登录、注册与找回密码页面

主要文件：

- `web/default/src/features/auth/auth-layout.tsx`
- `web/default/src/features/auth/sign-in/index.tsx`
- `web/default/src/features/auth/sign-in/components/user-auth-form.tsx`
- `web/default/src/features/auth/sign-up/index.tsx`
- `web/default/src/features/auth/sign-up/components/sign-up-form.tsx`
- `web/default/src/features/auth/forgot-password/`

### 5.1 当前问题

- 桌面端留白过大，表单缺少明确视觉容器。
- 与主页紫色科技视觉脱节。
- 主按钮仍为蓝色，品牌色不一致。
- Logo 和字标形式与主页不同。
- 登录、注册、找回密码之间缺少统一的状态反馈。
- 移动端表单宽度偏窄。
- `h-svh` 在开启 OAuth、验证码、协议内容后存在内容裁切风险。

### 5.2 推荐桌面布局

认证页调整为双栏布局：

```text
┌────────────────────────┬─────────────────────┐
│ 品牌视觉区 44%          │ 表单区域 56%         │
│                        │                     │
│ TUOSHANG AI            │ 欢迎回来             │
│ 简短产品价值说明        │ 输入框               │
│ 路由状态动态演示        │ 登录按钮             │
│ 安全/兼容/稳定标签      │ OAuth / Passkey      │
│                        │ 协议说明             │
└────────────────────────┴─────────────────────┘
```

- 左侧沿用主页网格和紫色光晕，但降低动效密度。
- 左侧展示 API Gateway Ready、Secure Access、Intelligent Routing 三个轻量状态。
- 右侧表单宽度保持 `420px–440px`。
- 表单使用低对比度卡片或玻璃边框，不使用厚重阴影。

### 5.3 移动端布局

- 隐藏桌面品牌视觉区，仅保留顶部品牌和主题/语言按钮。
- 表单宽度调整为 `calc(100% - 40px)`。
- 使用 `min-h-svh` 并允许页面滚动。
- 表单整体相对当前布局向上移动，减少中间大面积空白。
- 兼顾底部安全区和软键盘弹出后的可滚动空间。

### 5.4 输入框动画

- Focus：边框在 `180ms` 内过渡为品牌紫色。
- Focus 背景增加约 2% 的紫色透明度。
- 输入图标只改变颜色，不移动布局。
- 错误文本向下淡入，避免突然撑开页面。
- 可使用一次 `200ms`、位移不超过 `2px` 的轻微错误抖动。
- 不对输入框整体执行缩放，避免页面跳动。

### 5.5 按钮动画

- 默认使用品牌紫色渐变。
- Hover：上浮 `1px` 并增强阴影。
- Press：缩放至 `0.985`，持续 `80ms–100ms`。
- Loading 时按钮尺寸保持不变，文本淡出并显示 Spinner。
- 成功时 Spinner 平滑切换为 Check，约 `300ms` 后跳转。

### 5.6 登录页专项

- 增加“保持登录”选项。
- 密码显示按钮补充 Tooltip 和清晰的 `aria-label`。
- Passkey 作为高优先级次按钮展示。
- OAuth 按钮统一高度、边框、图标和文字基线。
- 登录失败后将焦点定位到对应错误字段。

### 5.7 注册页专项

- 增加密码规则实时状态：长度、字符要求、两次密码一致。
- 已满足条件使用逐条淡入的 Check 状态。
- 如果启用邮箱验证码，显示“账户信息 → 邮箱验证 → 完成”步骤条。
- 步骤切换使用约 `12px` 横向位移，不做整页翻转。

### 5.8 找回密码专项

- 邮件发送成功后切换为明确的完成状态，而不仅依赖 Toast。
- 展示目标邮箱和重新发送倒计时。
- 成功图标使用一次性描边动画。
- 保留明确的“返回登录”入口。

- 模型卡片信息层级调整为：模型名 → 供应商 → 能力标签 → 价格 → 可用状态。
- 价格和可用状态作为视觉焦点，降低非核心标签权重。

### 6.5 移动端布局

- 压缩标题区域高度。
- 搜索框和筛选按钮放在同一行。
- 使用底部 Sheet 展示完整筛选条件。
- 当前筛选项通过横向可滚动 Chips 展示。
- 模型卡片保持单列，价格和状态始终可见。

### 6.6 动画

- 模型卡片每行以 `40ms` 间隔错峰进入。
- 卡片/表格视图切换使用交叉淡化，不做大幅缩放。
- 筛选更新时旧内容淡出约 `100ms`，新内容淡入约 `180ms`。
- 桌面模型详情抽屉从右侧滑入，时长约 `320ms`。
- 移动端详情从底部弹出，时长约 `360ms`。
- 数字价格变化仅在真实数据变化时执行短暂滚动动画。

## 9. Dashboard 与后台页面

当前 Dashboard 已经使用 `motion/react`、`FadeIn`、Skeleton 和懒加载，基础动效体系相对完善。后续优化重点不是增加更多动画，而是统一品牌和交互规范。

建议：

- 将公共品牌色、圆角、阴影和状态色应用到统计卡片。
- 统一图表加载、空状态和错误状态。
- 数据刷新时保留图表容器高度，避免布局跳动。
- 数值更新使用短暂颜色过渡，不让整张卡片重新入场。
- 表格只在首次加载时执行错峰动画，分页和筛选后使用快速淡化。
- 抽屉、弹窗、筛选器使用统一的动效时长。
- 大数据量页面避免对每一行持续使用复杂动画。

## 10. 统一动效规范

| 场景 | 推荐时长 | 推荐动效 |
|---|---:|---|
| 按钮 Hover | 160–200ms | 颜色、阴影、上浮 1px |
| 按钮 Press | 80–120ms | 缩放至 0.985 |
| 输入框 Focus | 180ms | 边框和背景过渡 |
| 弹窗/抽屉 | 280–360ms | 位移 + 淡入 |
| 页面进入 | 350–450ms | `opacity + translateY(12px)` |
| 卡片错峰 | 40–80ms | 每行最多错峰 4–6 个 |
| Toast | 220–280ms | 右侧或底部滑入 |
| 导航收缩 | 600–700ms | 保留当前 Spring 风格 |
| Skeleton | 1.6–1.8s | 低对比度 Shimmer |

统一缓动：

```css
cubic-bezier(0.16, 1, 0.3, 1)
```

动效约束：

- 主要动画优先使用 `transform` 和 `opacity`。
- 避免对大量元素执行实时 `blur` 动画。
- 无限动画仅保留状态点、模型滚动和低速背景。
- 必须支持 `prefers-reduced-motion`。
- 表单错误和重要提示不能只依赖颜色或动画表达。
- 动画不得改变按钮和输入框的布局尺寸。

## 11. 设计 Token 建议

建议建立以下语义 Token：

```css
--brand-primary
--brand-secondary
--brand-accent
--brand-gradient
--surface-elevated
--surface-glass
--border-subtle
--border-interactive
--shadow-card
--shadow-floating
--motion-fast
--motion-default
--motion-slow
--ease-brand
```

组件只使用语义 Token，不在单独页面中重复定义接近但不同的紫色、阴影和动画时长。

## 13. 验收标准

### 视觉一致性

- 所有公共页面使用统一品牌标志、字标和主色。
- 登录注册与主页属于同一视觉体系。
- 主要 CTA、输入框和空状态样式统一。

### 响应式

- 从 `390px` 到 `1920px` 无横向溢出。
- 移动端软键盘弹出后表单仍可滚动和提交。
- 固定导航、抽屉和底部操作区不遮挡正文。

### 动效

- 页面和组件动画遵守统一时长与缓动。
- 开启减少动态效果后不保留非必要动画。
- 动画不造成 CLS、焦点丢失或交互阻塞。

### 可访问性

- 正文和按钮对比度满足 WCAG 2.1 AA。
- 所有图标按钮均有可访问名称。
- 错误、成功和禁用状态不只依赖颜色。
- 键盘焦点顺序与视觉顺序一致。

### 性能

- 首屏动画只使用合成属性。
- 长列表避免逐项复杂动画。
- 页面切换和筛选更新不重复加载无关大型模块。
